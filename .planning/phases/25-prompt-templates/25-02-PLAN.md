**12. build_platformer_game (PROMPT-03)**

- Description: `"Complete end-to-end workflow for building a 2D platformer game from empty project to playable prototype using MCP tools"`
- Arguments: `[{"name": "complexity", "description": "Complexity level: minimal (player+platforms), standard (enemies+collectibles+HUD), full (animations+sounds+menus) (default: standard)", "required": false}]`
- Lambda: Based on complexity, return numbered workflow steps:
  - Phase 1 -- Scene Setup: create_scene (Node2D root) -> save_scene
  - Phase 2 -- Player: create_character (type=2d, name=Player) -> write_script (player movement: gravity, jump, horizontal input, move_and_slide) -> attach_script
  - Phase 3 -- Level: create_node (StaticBody2D for ground) -> create_collision_shape (rectangle, ground-sized) -> create_node (Sprite2D or ColorRect for visual). For platforms: duplicate_node or create additional StaticBody2D nodes
  - Phase 4 -- Camera: create_node (Camera2D) -> set_node_property (set as current, set limits)
  - Phase 5 (standard+): Enemies with create_character, collectibles with create_node + Area2D + connect_signal
  - Phase 6 (standard+): HUD with create_ui_panel or create_node + set_layout_preset
  - Phase 7 (full): create_animation for walk/idle/jump cycles
  - Phase 8: run_game -> capture_game_viewport -> test with inject_input -> stop_game
  - `minimal`: Phases 1-4, 8 only
  - `standard`: Phases 1-6, 8
  - `full`: All phases
  - Each step must show the exact tool name and representative parameter JSON

**13. setup_tilemap_level (PROMPT-04)**
- Name: `"setup_tilemap_level"`
- Description: `"Step-by-step workflow for creating a TileMap-based level: tileset setup, painting terrain, adding collision, and level boundaries"`
- Arguments: `[{"name": "level_type", "description": "Level type: platformer_level, top_down_level, dungeon (default: platformer_level)", "required": false}]`
- Lambda: Based on level_type, return steps:
  - `platformer_level`: 1) get_scene_tree to see current structure, 2) create_node (TileMapLayer) as child of level container, 3) get_tilemap_info to verify setup, 4) set_tilemap_cells to paint ground row (e.g., y=10, x=0..19), 5) set_tilemap_cells to paint platforms at various heights, 6) set_tilemap_cells for walls/boundaries, 7) erase_tilemap_cells to create gaps/pits, 8) get_tilemap_cell_info to verify painted cells, 9) Save scene
  - `top_down_level`: Similar but paint full floor, walls around perimeter, obstacles inside, paths between areas
  - `dungeon`: Rooms as rectangular fills, corridors as 1-tile-wide paths, doors as single-tile gaps
  - Include notes on TileSet setup (source_id, atlas_coords) with get_tilemap_info
  - Reference tools: create_node, set_tilemap_cells, erase_tilemap_cells, get_tilemap_info, get_tilemap_cell_info, save_scene

**14. build_top_down_game (PROMPT-05)**
- Name: `"build_top_down_game"`
- Description: `"Complete end-to-end workflow for building a 2D top-down game from empty project to playable prototype using MCP tools"`
- Arguments: `[{"name": "genre", "description": "Sub-genre: adventure, shooter, rpg (default: adventure)", "required": false}]`
- Lambda: Based on genre, return numbered workflow:
  - Phase 1 -- Scene: create_scene (Node2D root) -> save_scene
  - Phase 2 -- Player: create_character (type=2d, name=Player) -> write_script (top-down 8-direction movement using Input.get_vector + move_and_slide) -> attach_script
  - Phase 3 -- World: create_node (TileMapLayer for terrain) -> set_tilemap_cells for ground -> set_tilemap_cells for walls/obstacles
  - Phase 4 -- Camera: create_node (Camera2D, set as current, enable smoothing)
  - Phase 5 -- Interaction:
    - adventure: NPCs (create_node Area2D + connect_signal for interaction)
    - shooter: Projectiles (write_script for bullet spawning + movement)
    - rpg: NPCs + items (create_node + connect_signal)
  - Phase 6 -- UI: create_ui_panel for HUD (health bar, score/inventory display)
  - Phase 7: run_game -> capture_game_viewport -> test -> stop_game
  - Each step: tool name + parameter JSON examples
