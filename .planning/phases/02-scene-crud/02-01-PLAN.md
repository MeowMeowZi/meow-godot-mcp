    **IMPORTANT: variant_parser has TWO layers -- one Godot-free for testing, one Godot-dependent for runtime.**

    The unit tests need to run WITHOUT Godot (same as test_protocol.cpp). Therefore, split the module:

    1. **Create `src/variant_parser.h`** with TWO function signatures:
       ```cpp
       #ifndef GODOT_MCP_MEOW_VARIANT_PARSER_H
       #define GODOT_MCP_MEOW_VARIANT_PARSER_H

       #include <nlohmann/json.hpp>
       #include <string>

       // Pure C++ parsing (no Godot dependency) -- for unit testing and pre-validation
       // Returns a JSON representation of the parsed value with type info
       // Used by unit tests and argument validation
       nlohmann::json parse_variant_hint(const std::string& value_str, const std::string& type_hint = "");

       #ifdef GODOT_MCP_MEOW_GODOT_ENABLED
       // Godot-dependent parsing -- used at runtime by scene_mutation.cpp
       #include <godot_cpp/variant/variant.hpp>
       #include <godot_cpp/classes/node.hpp>
       godot::Variant parse_variant(const std::string& value_str, godot::Node* node, const std::string& property);
       #endif

       #endif
       ```

    2. **Create `src/variant_parser.cpp`** with BOTH implementations:
       - `parse_variant_hint(value_str, type_hint)`: Pure C++ function that analyzes the string and returns JSON like `{"type": "vector2", "raw": "Vector2(100,200)"}` or `{"type": "color_hex", "raw": "#ff0000"}` or `{"type": "int", "value": 42}` or `{"type": "bool", "value": true}` or `{"type": "string", "value": "hello"}`. This tests the parsing LOGIC without Godot.
       - Detection rules for parse_variant_hint:
         a. If string matches `"null"` -> `{"type": "nil"}`
         b. If string matches regex-like pattern for Godot constructors (starts with uppercase letter followed by `(`) like `"Vector2("`, `"Color("`, `"Transform2D("`, etc. -> `{"type": "godot_constructor", "raw": value_str}` (str_to_var will handle at runtime)
         c. If string starts with `#` and length is 4, 5, 7, or 9 (hex color formats #RGB, #RGBA, #RRGGBB, #RRGGBBAA) and all chars after # are hex digits -> `{"type": "color_hex", "raw": value_str}`
         d. If type_hint is "bool" or string is exactly "true"/"false" -> `{"type": "bool", "value": true/false}`
         e. If type_hint is "int" or string is all digits (with optional leading minus) -> try `std::stoll`, return `{"type": "int", "value": N}`
         f. If type_hint is "float" or string matches number with decimal point -> try `std::stod`, return `{"type": "float", "value": N}`
         g. Fallback: `{"type": "string", "value": value_str}`

       - `parse_variant(value_str, node, property)` (Godot-dependent, guarded by `#ifdef GODOT_MCP_MEOW_GODOT_ENABLED`):
         a. Try `UtilityFunctions::str_to_var(String(value_str.c_str()))` -- if result is not NIL (or input was "null"), return it
         b. If starts with '#', try `Color::html_is_valid(String(value_str.c_str()))` then `Color::html(String(value_str.c_str()))`
         c. Get current property type: `Variant current = node->get(StringName(property.c_str())); Variant::Type target_type = current.get_type();`
         d. Switch on target_type: BOOL -> "true"/"false", INT -> std::stoll, FLOAT -> std::stod
         e. Fallback: return `Variant(String(value_str.c_str()))`
