    **ERR-08: Script parse error line information**

    Per RESEARCH.md: "GDScript::reload() returns Error enum but does NOT provide the parse error message through the godot-cpp API." The research recommends: "re-read the script source and do basic syntax validation (check for unterminated strings, mismatched brackets)."

    Two-pronged approach:

    **1. Add basic GDScript syntax validation to error_enrichment (pure C++, testable):**

    Add to error_enrichment.h:
    ```cpp
    struct ScriptErrorInfo {
        bool has_error;
        int line_number;        // 1-based, 0 if unknown
        std::string line_content;
        std::string error_description;
    };

    // Basic GDScript syntax check: finds common errors like
    // mismatched brackets/parens, unterminated strings, bad indentation
    ScriptErrorInfo check_gdscript_syntax(const std::string& source_code);
    ```

    Implement `check_gdscript_syntax` in error_enrichment.cpp:
    - Split source into lines
    - Track bracket/paren/brace nesting per line
    - Check for unterminated string literals (odd number of unescaped quotes on a line)
    - Check for common patterns: `func` not followed by space+name, `var` at wrong indent level
    - Return the first error found with line number and content
    - Bracket tracking:
      ```cpp
      int paren_depth = 0, bracket_depth = 0, brace_depth = 0;
      for (int i = 0; i < lines.size(); i++) {
          bool in_string = false;
          for (char c : lines[i]) {
              if (c == '"' && !in_string) in_string = true;
              else if (c == '"' && in_string) in_string = false;
              if (!in_string) {
                  if (c == '(') paren_depth++;
                  else if (c == ')') paren_depth--;
                  else if (c == '[') bracket_depth++;
                  else if (c == ']') bracket_depth--;
                  else if (c == '{') brace_depth++;
                  else if (c == '}') brace_depth--;
              }
              if (paren_depth < 0 || bracket_depth < 0 || brace_depth < 0) {
                  return {true, i + 1, lines[i], "Unmatched closing bracket/parenthesis"};
              }
          }
      }
      // At end, check if any are still open
      if (paren_depth != 0) return {true, last_paren_line, ..., "Unclosed parenthesis"};
      ```
    - Also check for unterminated multiline strings (triple-quote `"""`)
