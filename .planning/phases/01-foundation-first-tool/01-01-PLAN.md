  <name>Task 1: Initialize godot-cpp submodule, nlohmann/json, and SConstruct build system</name>
  <files>
    .gitmodules
    godot-cpp/ (submodule)
    thirdparty/nlohmann/json.hpp
    SConstruct

  <action>
1. Add godot-cpp as a git submodule pointing to master branch:
   ```
   git submodule add -b master https://github.com/godotengine/godot-cpp.git godot-cpp
   git submodule update --init --recursive
   ```

2. Download nlohmann/json 3.12.0 single header:
   ```
   mkdir -p thirdparty/nlohmann
   curl -L https://github.com/nlohmann/json/releases/download/v3.12.0/json.hpp -o thirdparty/nlohmann/json.hpp
   ```

3. Create `SConstruct` at project root. Model it after the godot-cpp-template SConstruct. It must:
   - Import the godot-cpp build environment from `godot-cpp/SConstruct`
   - Define a "gdextension" target that compiles all `.cpp` files in `src/` into a shared library
   - Output the library to `project/addons/godot_mcp_meow/bin/` with the naming convention: `libgodot_mcp_meow.{platform}.{target}.{arch}.{ext}` (e.g., `libgodot_mcp_meow.windows.template_debug.x86_64.dll`)
   - Add `thirdparty/` to the include path so `#include <nlohmann/json.hpp>` works from any source file
   - Add `src/` to the include path
   - Support `platform=` and `target=` arguments (inherited from godot-cpp env)
   - Define a "bridge" target (placeholder for Plan 03) that compiles `bridge/*.cpp` into a standalone executable. For now, just define the target structure -- the bridge source files don't exist yet, so guard it with a file existence check or comment it out
   - Default target should be the GDExtension library (not bridge)

Key SCons patterns from godot-cpp-template:
```python
env = SConscript("godot-cpp/SConstruct")
env.Append(CPPPATH=["src/", "thirdparty/"])
sources = Glob("src/*.cpp")
library = env.SharedLibrary(
    "project/addons/godot_mcp_meow/bin/libgodot_mcp_meow{}{}".format(
        env["suffix"], env["SHLIBSUFFIX"]
    ),
    source=sources,
)
Default(library)
```
  </action>

  </verify>
  <done>godot-cpp submodule initialized with recursive deps, nlohmann/json.hpp present in thirdparty/, SConstruct defines GDExtension build target with correct output path and include paths</done>

2. Create `src/register_types.cpp`:
   - Include `godot_cpp/core/class_db.hpp` and `godot_cpp/godot.hpp`
   - `initialize_mcp_module`: At `MODULE_INITIALIZATION_LEVEL_EDITOR`, do nothing yet (MCPPlugin class comes in Plan 02)

   - Follow the exact pattern from research Code Examples (Pattern 3)

3. Create `project/addons/godot_mcp_meow/godot_mcp_meow.gdextension`:
   ```ini
   [configuration]
   entry_symbol = "mcp_library_init"
   compatibility_minimum = "4.3"
   reloadable = true

   [libraries]
   linux.debug.x86_64 = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.linux.template_debug.x86_64.so"
   windows.debug.x86_64 = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.windows.template_debug.x86_64.dll"
   macos.debug = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.macos.template_debug.framework"
   linux.release.x86_64 = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.linux.template_release.x86_64.so"
   windows.release.x86_64 = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.windows.template_release.x86_64.dll"
   macos.release = "res://addons/godot_mcp_meow/bin/libgodot_mcp_meow.macos.template_release.framework"
   ```

4. Create `project/addons/godot_mcp_meow/bin/.gitkeep` (empty file to track the bin directory in git).
