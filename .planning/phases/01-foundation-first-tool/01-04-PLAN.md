    **Step 1: Open test project in Godot**
    1. Open Godot 4.3+ editor
    2. Open the project at `project/project.godot`
    3. Check the Output panel for "MCP Meow: Server started on port 9900" message
    4. If no message appears, check Project Settings > Plugins to see if the GDExtension loaded

    **Step 2: Create a test scene**
    1. Create a new scene (Scene > New Scene)
    2. Add a root Node2D named "TestRoot"
    3. Add a child Sprite2D named "PlayerSprite" under TestRoot
    4. Add a child Camera2D named "MainCamera" under TestRoot
    5. Save the scene as `test_scene.tscn`

    **Step 3: Test MCP handshake via bridge**
    1. Open a terminal/command prompt
    2. Run the bridge: `project\addons\godot_mcp_meow\bin\godot-mcp-bridge.exe --port 9900`
    3. The bridge should connect (check stderr for "Connected to..." message)
    4. Type the initialize request (paste as a single line, press Enter):
       ```
       {"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"manual-test","version":"1.0"}}}
       ```
    5. Expected: stdout shows response with `"protocolVersion":"2025-03-26"` and `"serverInfo":{"name":"godot-mcp-meow"}`

    **Step 4: Test tools/list**
    1. Type (paste as single line, press Enter):
       ```
       {"jsonrpc":"2.0","method":"notifications/initialized"}
       ```
    2. No response expected (notification)
    3. Type:
       ```
       {"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}
       ```
    4. Expected: response with tools array containing `"name":"get_scene_tree"`

    **Step 5: Test get_scene_tree**
    1. Type:
       ```
       {"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"get_scene_tree","arguments":{}}}
       ```
    2. Expected: response containing scene tree with TestRoot (Node2D), PlayerSprite (Sprite2D), and MainCamera (Camera2D)
    3. Verify the JSON includes name, type, path, and children fields

    **Step 6: Verify Godot editor remains responsive**
    1. While bridge is connected, interact with the Godot editor
    2. Move nodes, resize windows, switch tabs
    3. Editor should remain fully responsive (no freezing or lag)
