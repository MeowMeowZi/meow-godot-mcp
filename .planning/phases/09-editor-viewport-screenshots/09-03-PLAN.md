4. **test_image_content_has_type_image** -- Verify response content item has `"type": "image"` (not "text") (VWPT-03)
5. **test_image_content_has_mimetype** -- Verify response content item has `"mimeType": "image/png"` (VWPT-03)
6. **test_base64_decodes_to_png** -- Decode `content[0]["data"]` from base64, verify first 8 bytes match PNG signature `\x89PNG\r\n\x1a\n` (VWPT-03)
7. **test_metadata_text_content** -- Verify `content[1]["type"] == "text"` and its text parses as JSON with `viewport_type`, `width`, `height` keys
8. **test_capture_with_resize** -- `capture_viewport` with `{"viewport_type": "2d", "width": 640, "height": 480}` returns image; parse metadata to confirm width==640 and height==480
9. **test_invalid_viewport_type** -- `capture_viewport` with `{"viewport_type": "invalid"}` returns response where `content[0]["type"] == "text"` and parsed text contains `"error"` key
10. **test_capture_default_no_args** -- `capture_viewport` with no arguments captures 2D viewport (default); response has `content[0]["type"] == "image"`

- The MCPClient.send() method sends JSON-RPC `tools/call` with `{"name": tool_name, "arguments": args}`.
- For viewport tests, the response uses `content` array directly (not wrapped in text like other tools).
- Success responses have `content[0]["type"] == "image"` -- this distinguishes ImageContent from TextContent.
- Error responses still have `content[0]["type"] == "text"` (using create_tool_result for errors).
- Import `base64` module for PNG signature validation in test 6.
- Use `import json` to parse metadata text in test 7.
- The script docstring should list all test names and requirement coverage.
- Print pass/fail per test with test name, final summary with counts.
