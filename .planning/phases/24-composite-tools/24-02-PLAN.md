    **Header additions (src/composite_tools.h):**
    Add three new Godot-guarded function declarations inside the `#ifdef MEOW_GODOT_MCP_GODOT_ENABLED` block:

    ```cpp
    // COMP-03: Create complete character (CharacterBody + CollisionShape + visual)
    nlohmann::json create_character(const std::string& name, const std::string& char_type,
                                     const std::string& shape_type, const std::string& parent_path,
                                     const std::string& sprite_texture,
                                     const std::string& script_template,
                                     godot::EditorUndoRedoManager* undo_redo);

    // COMP-04: Create UI panel from declarative JSON spec
    nlohmann::json create_ui_panel(const nlohmann::json& spec, const std::string& parent_path,
                                    godot::EditorUndoRedoManager* undo_redo);

    // COMP-05: Deep-copy node subtree to new parent
    nlohmann::json duplicate_node(const std::string& source_path, const std::string& target_parent_path,
                                   const std::string& new_name,
                                   godot::EditorUndoRedoManager* undo_redo);
    ```

    **Implementation (src/composite_tools.cpp):**

    **create_character:**
    1. Get scene root, validate it exists.
    2. Find parent node (or use scene root if parent_path is empty).
    3. Determine dimension: `char_type` is "2d" or "3d". Map to class names: "CharacterBody2D"/"CharacterBody3D".
    4. Begin single UndoRedo action: `create_action("MCP: Create Character")`
    5. Create CharacterBody node: `ClassDB::instantiate(StringName(body_class))`, set name, add_child to parent via undo_redo, set_owner to scene_root.
    6. Create CollisionShape node: Instantiate CollisionShape2D/3D, create default shape resource (RectangleShape2D for 2D with size Vector2(30,30), or CapsuleShape3D for 3D with radius 0.5 height 2.0). If `shape_type` is specified, use the shape_types map from physics_tools.cpp pattern (rectangle/circle/capsule/box/sphere/capsule_3d). Set shape on collision node. Add as child of CharacterBody via undo_redo.
    7. If `sprite_texture` is non-empty: Create Sprite2D/Sprite3D node, load texture via `ResourceLoader::load()`, set texture property, add as child of CharacterBody via undo_redo.
    8. If `script_template` == "basic_movement": Create a GDScript with:
       - For 2D: `extends CharacterBody2D\n\nconst SPEED = 300.0\nconst JUMP_VELOCITY = -400.0\n\nfunc _physics_process(delta: float) -> void:\n\tvar velocity_x = Input.get_axis("ui_left", "ui_right") * SPEED\n\tvelocity.x = velocity_x\n\tif not is_on_floor():\n\t\tvelocity += get_gravity() * delta\n\tif Input.is_action_just_pressed("ui_accept") and is_on_floor():\n\t\tvelocity.y = JUMP_VELOCITY\n\tmove_and_slide()\n`
       - For 3D: `extends CharacterBody3D\n\nconst SPEED = 5.0\nconst JUMP_VELOCITY = 4.5\n\nfunc _physics_process(delta: float) -> void:\n\tif not is_on_floor():\n\t\tvelocity += get_gravity() * delta\n\tvar input_dir = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")\n\tvar direction = (transform.basis * Vector3(input_dir.x, 0, input_dir.y)).normalized()\n\tif direction:\n\t\tvelocity.x = direction.x * SPEED\n\t\tvelocity.z = direction.z * SPEED\n\telse:\n\t\tvelocity.x = move_toward(velocity.x, 0, SPEED)\n\t\tvelocity.z = move_toward(velocity.z, 0, SPEED)\n\tif Input.is_action_just_pressed("ui_accept") and is_on_floor():\n\t\tvelocity.y = JUMP_VELOCITY\n\tmove_and_slide()\n`
       - Use the manual GDScript construction pattern (from MEMORY.md -- ResourceLoader::load crashes on newly created files): `Ref<GDScript> script; script.instantiate(); script->set_source_code(String(code.c_str())); script->set_path(String(path.c_str())); script->reload();` then set script on CharacterBody node.
    9. `commit_action()`
    10. Return `{"success": true, "root_path": "...", "nodes_created": ["CharacterBody", "CollisionShape", "Sprite2D"], "script": "res://..."}` or `{"error": "..."}`.

    **create_ui_panel:**
    1. Get scene root, validate.
    2. Parse spec JSON. Required fields: `root_type` (string, e.g., "PanelContainer", "VBoxContainer", "HBoxContainer"). Optional: `children` (array), `style` (object), `name` (string).
    3. Validate `root_type` is a valid Control subclass. If not, return error.
    4. Begin single UndoRedo action: `create_action("MCP: Create UI Panel")`
    5. Create root container node with name (default to root_type if no name given). Add to parent via undo_redo.
    6. If `style` object present: After adding root, apply StyleBoxFlat. Create StyleBoxFlat resource, set properties from style object (bg_color as Color, corner_radius as int, border_width as int, border_color as Color, content_margin_* as int). Set as theme override "panel" on root node via undo_redo add_do_method.
    7. For each child in `children` array: Each child is `{type: "Label", name: "Title", text: "Hello", ...}`. Create node of given type, set name, add as child of root via undo_redo. For known properties like `text`, `placeholder_text`, `min_value`, `max_value`, `value`, `button_pressed`, `alignment`: set via add_do_method(child_node, "set", property, value). For `theme_overrides` sub-object on child: apply font_size, font_color etc.
    8. Maximum two levels (root + children). If a child has `children`, skip with warning in response.
    9. `commit_action()`
    10. Return `{"success": true, "root_path": "...", "children_created": N}` or `{"error": "..."}`.

    **duplicate_node:**
    1. Get scene root, validate.
    2. Find source node by `source_path`. If not found, return error.
    3. Find target parent by `target_parent_path`. If not found, return error.
    4. Call `source_node->duplicate()` which deep-copies the entire subtree (Godot built-in).
    5. If `new_name` is non-empty, set name on the duplicated root.
    6. Begin single UndoRedo action: `create_action("MCP: Duplicate Node")`
    7. Add duplicated node to target parent via undo_redo. Recursively set owner on all descendants via a helper `set_owner_recursive(Node* node, Node* owner)` that iterates children and calls `add_do_method(child, "set_owner", owner)`.
    8. `commit_action()`
    9. Return `{"success": true, "source_path": "...", "new_path": "...", "nodes_duplicated": N}` where N = total node count in subtree.

    **Important implementation notes:**
    - Do NOT call existing `create_node()` or `create_collision_shape()` functions because they each create their own UndoRedo actions. We need a SINGLE action for the entire composite. Instead, directly use `ClassDB::instantiate()` + `add_do_method(parent, "add_child", ...)` pattern from physics_tools.cpp.
    - For script attachment in create_character, use the manual GDScript construction (instantiate + set_source_code + set_path + reload) per the known bug documented in MEMORY.md.
    - For create_ui_panel style, create StyleBoxFlat inline (not via create_stylebox function which has its own UndoRedo).

    if (tool_name == "create_character") {
        auto& args = get_args(params);

        std::string shape_type = get_string(args, "shape_type");
