**Two approaches are available for intercepting print() output. Use APPROACH A (game-side forwarding) as the primary mechanism:**

**APPROACH A: Game-side print output forwarding via meow_mcp_bridge.gd**

In `meow_mcp_bridge.gd`, in `_ready()` after registering the message capture:
- Store initial timestamp: `var _start_time := Time.get_ticks_msec()`
- Call `_start_log_capture()` which does nothing explicit — the real capture happens via Godot's `print` interception

Since Godot 4.x does not expose a direct `print()` hook in GDScript, use the **custom logger** approach:
- In `_ready()`, add self as a logger via `_setup_log_forwarding()`
- Override print capture by hooking into `_process()` to periodically check `EngineDebugger` for queued output — BUT this is not how Godot works.

**REVISED APPROACH: The editor-side already receives all print output.** Godot's EditorDebuggerPlugin receives `output` messages automatically from the running game. The prefix for these messages is `"output"`. The `_capture()` callback receives messages with prefix matching what `_has_capture()` returns true for.

So the correct approach is:

1. **In `game_bridge.h`**: Add a `_has_capture` that also accepts `"output"` prefix. Godot's debug protocol sends `output:N` messages (where N is a sequence number or just `output`) for all print/push_error/push_warning output from the game process.

   Actually, looking at Godot source code, `_has_capture` is called with the prefix before the colon. The engine sends messages like `"output:..."`. So `_has_capture("output")` should return true.

   Wait — rethinking. The Godot editor's built-in Output panel already captures these. The EditorDebuggerPlugin's `_has_capture` is called to see if THIS plugin wants to handle messages with that prefix. If we return true for "output", we intercept ALL output messages. But the built-in editor panel might stop receiving them.

   **Better approach: Use the companion GDScript to forward print output explicitly.**

   In `meow_mcp_bridge.gd`:
   - Add a custom `_log_message(message: String, type: String)` function
   - Use `EngineDebugger.send_message("meow_mcp:log_output", [message, type, Time.get_ticks_msec()])` to send each log line to the editor
   - Redirect print output by calling `print_rich` or other approach... No, GDScript cannot intercept print().

   **ACTUAL CORRECT APPROACH: Intercept "output" messages in _has_capture/_capture on the editor side.**

   Godot's editor debugger dispatches `output` messages to plugins. Multiple plugins CAN capture the same prefix — `_has_capture` returning true just means this plugin wants to SEE those messages. The built-in editor Output panel is NOT an EditorDebuggerPlugin, it handles output separately. So intercepting "output" in our plugin will NOT prevent the Output panel from showing print output.

   In Godot's source, `EditorDebuggerNode::_debugger_data` handles output messages separately from `EditorDebuggerSession::_capture`. The `_capture` system is for EditorDebuggerPlugin extensions. Standard output messages (print, errors, warnings) are sent as `output` messages and can be captured by plugins without blocking the normal Output panel.

```cpp
// At the beginning of _capture, handle output messages
// The prefix for output is "output" — p_message will be like "output:0", "output:1", etc.
// But _has_capture returns true for "output" prefix, so messages starting with "output:" are routed here.
// Actually, the full p_message might just be "output" with data containing the text.
// Need to check: Godot's RemoteDebugger sends messages like "output" with data array.

// Check if this is an output capture (not meow_mcp)
if (p_message.begins_with("output")) {
    // Godot output message: data[0] = Array of strings, data[1] = Array of types
    // OR data[0] = message string, data[1] = type int
    // The exact format depends on Godot version. For Godot 4.3+:
    // p_data contains pairs: [text, type, text, type, ...]
    // Actually in Godot 4.x, debugger sends "output" with data = [strings_array, types_array]
    // But through _capture interface, p_data has the actual arrays.

    // Parse the output data and buffer it
    if (p_data.size() >= 2) {
        // Try as individual text+type pair first
        if (p_data[0].get_type() == Variant::STRING) {
            String text = p_data[0];
            int type_val = p_data[1];
            std::string level = "info";
            if (type_val == 1) level = "error";
            else if (type_val == 2) level = "warning";

            std::string msg(text.utf8().get_data());
            // Remove trailing newline if present
            if (!msg.empty() && msg.back() == '\n') msg.pop_back();
            if (!msg.empty()) {
                auto now = std::chrono::steady_clock::now().time_since_epoch();
                int64_t ms = std::chrono::duration_cast<std::chrono::milliseconds>(now).count();
                log_buffer.push_back({msg, level, ms});
            }
        }
        // Also try as arrays (Godot sometimes sends arrays of output)
        else if (p_data[0].get_type() == Variant::PACKED_STRING_ARRAY || p_data[0].get_type() == Variant::ARRAY) {
            // Handle array format
            Array texts = p_data[0];
            Array types = p_data[1];
            auto now = std::chrono::steady_clock::now().time_since_epoch();
            int64_t ms = std::chrono::duration_cast<std::chrono::milliseconds>(now).count();
            for (int i = 0; i < texts.size(); i++) {
                String text = texts[i];
                int type_val = (i < types.size()) ? (int)types[i] : 0;
                std::string level = "info";
                if (type_val == 1) level = "error";
                else if (type_val == 2) level = "warning";
                std::string msg(text.utf8().get_data());
                if (!msg.empty() && msg.back() == '\n') msg.pop_back();
                if (!msg.empty()) {
                    log_buffer.push_back({msg, level, ms});
                }
            }
        }
    }
    return true;  // We handled the output message
}
```

```cpp
nlohmann::json MeowDebuggerPlugin::get_buffered_game_output(
    bool clear_after_read, const std::string& level_filter,
    int64_t since_ms, const std::string& keyword) {

    nlohmann::json lines = nlohmann::json::array();
    size_t start = static_cast<size_t>(log_buffer_read_pos);

    for (size_t i = start; i < log_buffer.size(); i++) {
        const auto& entry = log_buffer[i];

        // Filter by level
        if (!level_filter.empty() && entry.level != level_filter) continue;

        // Filter by time
        if (since_ms > 0 && entry.timestamp_ms < since_ms) continue;

        // Filter by keyword
        if (!keyword.empty() && entry.message.find(keyword) == std::string::npos) continue;

        lines.push_back({
            {"text", entry.message},
            {"level", entry.level},
            {"timestamp_ms", entry.timestamp_ms}
        });
    }

    if (clear_after_read) {
        log_buffer_read_pos = static_cast<int64_t>(log_buffer.size());
    }

    bool game_running_val = game_connected;

    return {
        {"success", true},
        {"lines", lines},
        {"count", lines.size()},
        {"total_buffered", log_buffer.size()},
        {"game_running", game_running_val}
    };
}

void MeowDebuggerPlugin::clear_log_buffer() {
    log_buffer.clear();
    log_buffer_read_pos = 0;
}
```
