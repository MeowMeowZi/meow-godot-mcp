    - Pure C++ (no Godot): `std::string truncate_script_source(const std::string& source, int line_count)` — if line_count > 100, return first 50 lines + truncation notice; else return source unchanged
    - Under `#ifdef MEOW_GODOT_MCP_GODOT_ENABLED`:
      - `nlohmann::json get_enriched_scene_tree()` — traverses scene tree with depth=3, enriches each node with: script_path + script_source (truncated) + script_line_count, signal connections (outgoing from this node + incoming to this node), @export properties + transform + visibility. Enforces 10KB response size limit: if serialized JSON exceeds 10240 bytes, return summary (node count, root info) with message "Response exceeds 10KB limit. Use godot://node/{path} resource templates to query specific nodes."
      - `nlohmann::json get_enriched_project_files()` — calls collect_files pattern but adds: file_size via FileAccess::get_length(), file_type via classify_file_type(extension), modified_time via FileAccess::get_modified_time() (returns Unix timestamp)

    Create src/resource_tools.cpp implementing all functions:
    - classify_file_type: static const map from extension to type string. Extensions: .tscn/.scn -> "scene", .gd -> "script", .tres/.res -> "resource", .png/.jpg/.jpeg/.svg/.webp -> "image", .ogg/.wav/.mp3 -> "audio", everything else -> "other"
    - truncate_script_source: split by newlines, if count > 100, keep first 50, append notice line
    - get_enriched_scene_tree:
      1. Get edited scene root via EditorInterface::get_singleton()->get_edited_scene_root()
      2. If null, return {"tree": null, "message": "No scene currently open"}
      3. Recursive traversal (max depth 3): for each node, build JSON with name, type, path, then:
         a. Script: check node->get_script(), if valid Ref<Script>, include script_path, read file via FileAccess for source, apply truncate_script_source, include line_count
         b. Signals: call node->get_signal_connection_list() for each signal in node->get_signal_list() to get outgoing connections; also iterate all scene tree nodes to find incoming connections (signals connected TO this node). Format: {"outgoing": [{"signal": "pressed", "target": "Panel/Label", "method": "_on_pressed"}], "incoming": [{"source": "Panel/Button", "signal": "pressed", "method": "_on_button_pressed"}]}
         c. Properties: Node2D/Node3D transform (position/rotation/scale), CanvasItem/Node3D visibility, @export properties (iterate property list, check PROPERTY_USAGE_SCRIPT_VARIABLE flag)
      4. After serialization, check JSON dump size. If > 10240 bytes, return summary instead
    - get_enriched_project_files:
      1. Recursive directory traversal from res:// (same pattern as existing collect_files)
      2. For each file: path, extension, type (classify_file_type), size (FileAccess::get_length after open READ), modified_time (FileAccess::get_modified_time)
      3. Return {"success": true, "files": [...], "count": N}
    - enrich_node_detail: full single-node detail (for Plan 02 URI templates):
      1. Resolve node from path via EditorInterface
      2. All properties via get_property_list (STORAGE+EDITOR usage, skip INTERNAL)
      3. Full script source (no truncation)
      4. Child list (one level): name + type for each child
      5. All signal connections (outgoing + incoming)

    - Test truncate_script_source with long content (>100 lines) returns first 50 + notice

    Update tests/CMakeLists.txt: add test_resource_tools executable linking resource_tools.cpp (pure C++ parts only, no Godot).

    IMPORTANT: The incoming signal detection requires iterating the scene tree to find connections targeting the current node. For the depth-limited tree traversal, collect incoming connections by doing a full tree walk once at the start (build a map of node_path -> incoming_connections), then attach during node serialization.

    IMPORTANT: For @export properties, check PROPERTY_USAGE_SCRIPT_VARIABLE flag on each property. This flag is set for variables declared with @export in GDScript.
