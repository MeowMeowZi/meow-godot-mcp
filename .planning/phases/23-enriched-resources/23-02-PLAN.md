    2. Expand the "resources/read" handler to support URI template matching. After the existing exact-match checks for "godot://scene_tree" and "godot://project_files", add prefix-based matching:

       a. godot://node/{path} — extract path after "godot://node/" prefix:
          ```cpp
          const std::string node_prefix = "godot://node/";
          if (uri.substr(0, node_prefix.size()) == node_prefix) {
              std::string node_path = uri.substr(node_prefix.size());
              if (node_path.empty()) {
                  return mcp::create_error_response(id, mcp::INVALID_PARAMS, "Missing node path in URI. Example: godot://node/Player/Sprite2D");
              }
              nlohmann::json detail = enrich_node_detail(node_path);
              nlohmann::json contents = nlohmann::json::array();
              contents.push_back({{"uri", uri}, {"mimeType", "application/json"}, {"text", detail.dump()}});
              return mcp::create_resource_read_response(id, contents);
          }
          ```

       b. godot://script/{path} — extract res:// path after "godot://script/" prefix:
          ```cpp
          const std::string script_prefix = "godot://script/";
          if (uri.substr(0, script_prefix.size()) == script_prefix) {
              std::string script_path = uri.substr(script_prefix.size());
              if (script_path.empty()) {
                  return mcp::create_error_response(id, mcp::INVALID_PARAMS, "Missing script path in URI. Example: godot://script/res://player.gd");
              }
              nlohmann::json result = read_script(script_path);
              nlohmann::json contents = nlohmann::json::array();
              contents.push_back({{"uri", uri}, {"mimeType", "application/json"}, {"text", result.dump()}});
              return mcp::create_resource_read_response(id, contents);
          }
          ```

       c. godot://signals/{path} — extract node path after "godot://signals/" prefix:
          ```cpp
          const std::string signals_prefix = "godot://signals/";
          if (uri.substr(0, signals_prefix.size()) == signals_prefix) {
              std::string sig_node_path = uri.substr(signals_prefix.size());
              if (sig_node_path.empty()) {
                  return mcp::create_error_response(id, mcp::INVALID_PARAMS, "Missing node path in URI. Example: godot://signals/Player");
              }
              nlohmann::json result = get_node_signals(sig_node_path);
              nlohmann::json contents = nlohmann::json::array();
              contents.push_back({{"uri", uri}, {"mimeType", "application/json"}, {"text", result.dump()}});
              return mcp::create_resource_read_response(id, contents);
          }
          ```

    3. Keep the existing fallback error for unknown URIs at the end.

    4. Build with scons to verify everything compiles: `scons target=template_debug -j8`

    IMPORTANT: The URI prefix matching must come AFTER the exact matches for "godot://scene_tree" and "godot://project_files" to avoid false matches. The prefix strings do NOT overlap with the exact-match URIs (node/ vs scene_tree, script/ vs project_files), so ordering is safe but keep it clean.

    IMPORTANT: For godot://script/{path}, the {path} value will be something like "res://player.gd" — a res:// path. The read_script function already validates res:// prefix internally, so just pass it through.

    IMPORTANT: For godot://node/{path} and godot://signals/{path}, the {path} value is a scene-relative node path like "Player" or "Player/Sprite2D" — same format as used by get_scene_tree tool's root_path parameter.
