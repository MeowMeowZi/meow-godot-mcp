**2. set_theme_override (UISYS-02):**

Override type detection heuristic:
```cpp
// Known color keys (exact match or suffix _color)
// Known font_size keys (exact match "font_size" or suffix _size)
// Known constant keys: "separation", "h_separation", "v_separation", "outline_size", etc.
// Known stylebox keys: "normal", "hover", "pressed", "disabled", "focus", "panel", etc.
// Fallback: try to detect from value format
```

Concrete detection logic:
- If key ends with `_color` OR key is exactly `"font_color"`, `"icon_color"`: color type
- If key is exactly `"font_size"` OR ends with `_size` (but NOT `_color`): font_size type (integer)
- If key is `"separation"`, `"h_separation"`, `"v_separation"`, `"outline_size"`, `"margin_left"`, `"margin_top"`, `"margin_right"`, `"margin_bottom"`, `"icon_max_width"`: constant type (integer)
- Fallback: try to parse value as color (if hex or Color()). If that works, treat as color. Otherwise treat as constant (integer parse).

Implementation:
- Validate `overrides` is a JSON object
- `undo_redo->create_action(String("MCP: Set theme overrides"))` -- single undo action for entire batch
- `ctrl->begin_bulk_theme_override()`
- For each key-value pair, detect type and:
  - Color: parse color, `add_do_method(ctrl, "add_theme_color_override", key, color)`, capture old with `has_theme_color_override`/`get_theme_color` for undo (or `remove_theme_color_override` if no old override)
  - Font size: parse int, `add_do_method(ctrl, "add_theme_font_size_override", key, size)`, similar undo
  - Constant: parse int, `add_do_method(ctrl, "add_theme_constant_override", key, value)`, similar undo
- `ctrl->end_bulk_theme_override()`
- `undo_redo->commit_action()`
- Return `{"success", true}` with count of applied overrides
