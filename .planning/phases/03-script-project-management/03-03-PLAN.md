<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: End-to-end UAT of all Phase 3 tools</name>
  <what-built>
    Phase 3 adds 8 new MCP tools (read_script, write_script, edit_script, attach_script, detach_script, list_project_files, get_project_settings, get_resource_info), 2 MCP Resources (godot://scene_tree, godot://project_files), and IO thread architecture. Total: 12 tools, 2 resources.
  </what-built>
  <how-to-verify>
    **Prerequisites:** Open the Godot project at `project/` with the plugin enabled. Connect an MCP client (Claude Desktop or similar) via the bridge executable.

    **Test 1 - SCRP-01: read_script**
    Call `read_script` with path `res://test_root.gd` (or any existing .gd file).
    Expected: Returns file content as string with line_count.

    **Test 2 - SCRP-02: write_script**
    Call `write_script` with path `res://test_new_script.gd` and content `extends Node\n\nfunc _ready():\n\tprint("hello")\n`.
    Expected: Returns success. File appears in FileSystem dock. Calling again on same path returns error (file already exists).

    **Test 3 - SCRP-03: edit_script (insert)**
    Call `edit_script` with path `res://test_new_script.gd`, operation `insert`, line `3`, content `\tvar x = 42`.
    Expected: Returns success. Reading file shows new line 3 is `\tvar x = 42`.

    **Test 4 - SCRP-03: edit_script (replace)**
    Call `edit_script` with path `res://test_new_script.gd`, operation `replace`, line `3`, content `\tvar x = 99`.
    Expected: Returns success. Line 3 now reads `\tvar x = 99`.

    **Test 5 - SCRP-03: edit_script (delete)**
    Call `edit_script` with path `res://test_new_script.gd`, operation `delete`, line `3`.
    Expected: Returns success. Line count decreases by 1.

    **Test 6 - SCRP-04: attach_script**
    Create a Node2D in the scene. Call `attach_script` with the node path and `res://test_new_script.gd`.
    Expected: Node shows script icon in scene tree. Inspector shows the script. Ctrl+Z undoes attachment.

    **Test 7 - SCRP-04: detach_script**
    Call `detach_script` on the same node.
    Expected: Script icon disappears. Ctrl+Z re-attaches the script.

    **Test 8 - PROJ-01: list_project_files**
    Call `list_project_files` with no arguments.
    Expected: Returns array of {path, type} objects covering project files.

    **Test 9 - PROJ-02: get_project_settings**
    Call `get_project_settings` with no arguments.
    Expected: Returns structured settings including application/config/name.

    **Test 10 - PROJ-03: get_resource_info**
    Call `get_resource_info` with path to a .tres or .tscn file in the project.
    Expected: Returns resource type and property names/values.

    **Test 11 - PROJ-04: resources/list**
    Send a `resources/list` MCP method call.
    Expected: Returns 2 resources: godot://scene_tree and godot://project_files.

    **Test 12 - PROJ-04: resources/read**
    Send `resources/read` with uri `godot://scene_tree`.
    Expected: Returns scene tree JSON in contents array.
    Send `resources/read` with uri `godot://project_files`.
    Expected: Returns file list JSON in contents array.

    **Test 13 - MCP-04: IO thread**
    Verify the Godot output log shows "IO thread started" on plugin load.
    Verify multiple rapid tool calls work without blocking the editor.

    **Test 14 - Regression: Scene CRUD still works**
    Call `get_scene_tree`, `create_node`, `set_node_property`, `delete_node` to verify Phase 2 tools are unbroken.

    **Cleanup:** Delete test_new_script.gd from the project after testing.
  </how-to-verify>
